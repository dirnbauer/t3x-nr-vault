Quick Start

Store a Secret

Navigate to Secrets and click the + button in the header to create a new secret. Provide an identifier (e.g., stripe_api_key) and the secret value.

Use in Configuration

Reference secrets in your TYPO3 configuration using the vault syntax:

%vault(stripe_api_key)%

Usage Examples

Context Example
Site Configuration apiSecret: '%vault(site_api_secret)%'
In config/sites/*/config.yaml
TypoScript lib.auth.value = Bearer %vault(auth_token)%
Requires frontend_accessible flag + cache disabled
PHP Code $this->vaultService->retrieve('my_api_key')
Inject VaultServiceInterface via constructor
CLI bin/typo3 vault:store my_key "secret_value"
bin/typo3 vault:retrieve my_key
bin/typo3 vault:list

CLI Commands

Command Description
vault:initInitialize vault and generate master key
vault:storeStore a new secret
vault:retrieveRetrieve a secret value
vault:listList all secrets
vault:rotateRotate a secret's value
vault:deleteDelete a secret
vault:auditView audit log entries
vault:scanScan for plaintext secrets in database
vault:migrate-fieldMigrate a database field to vault
vault:rotate-master-keyRotate the master encryption key
vault:cleanup-orphansRemove orphaned secrets
Run bin/typo3 vault:<command> --help for detailed usage.

Security Features

Envelope Encryption

Each secret is encrypted with its own unique Data Encryption Key (DEK), which is then encrypted with the master key.

Access Control

Secrets can be restricted to specific backend users or groups. Every access is logged in the audit trail.

Audit Logging

Tamper-evident audit chain records all secret operations with cryptographic integrity verification.
