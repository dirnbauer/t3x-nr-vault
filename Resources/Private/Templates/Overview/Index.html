Quick Start

Store a Secret

Navigate to Secrets and click the + button in the header to create a new secret. Provide an identifier (e.g., stripe_api_key) and the secret value.

Use in Configuration

Reference secrets in your TYPO3 configuration using the vault syntax:

%vault(stripe_api_key)%

When to Use What

Site-Specific API Keys

Payment gateways, external services, or any secret that differs per site. Resolved at runtime from config/sites/*/config.yaml.

settings:
  stripeKey: '%vault(stripe_api_key)%'
  mailchimpKey: '%vault(mailchimp_key)%'

Per-Record Credentials

Let editors manage secrets per record (e.g., per-integration or per-client settings). Values encrypted automatically via TCA.

'api_key' => [
    'label' => 'Client API Key',
    'config' => [
        'type' => 'input',
        'renderType' => 'vaultSecret',
    ],
],

Extension Services

When your extension needs secrets programmatically - API clients, payment processors, external integrations.

public function __construct(
    private readonly VaultServiceInterface $vault,
) {}

public function callExternalApi(): Response
{
    $apiKey = $this->vault->retrieve('ext_api_key');
    // use $apiKey...
}

HTTP Client with Secrets

PSR-18 compatible API calls with automatic secret injection. Supports Bearer, API-Key, Basic Auth, OAuth2, and more.

public function __construct(
    private readonly VaultHttpClientInterface $http,
) {}

public function fetchData(): array
{
    $client = $this->http->withAuthentication(
        'api_token',
        SecretPlacement::Bearer,
    );
    $request = new Request('GET', 'https://api.example.com/data');
    $response = $client->sendRequest($request);
    return json_decode($response->getBody()->getContents(), true);
}

Deployment & CI/CD

Automate secret provisioning in deployment pipelines, or manage secrets via scripts without backend access.

# In deployment script
bin/typo3 vault:store smtp_password "$SMTP_PASS"
bin/typo3 vault:store api_key "$API_KEY"

# Rotate on schedule
bin/typo3 vault:rotate api_key "$NEW_KEY"

Frontend Rendering (use sparingly)

When secrets must appear in rendered output (e.g., JS API keys). Requires frontend_accessible flag and cache disabled.

lib.mapsKey = TEXT
lib.mapsKey.value = %vault(google_maps_key)%
lib.mapsKey.stdWrap.cache.disable = 1

TCA Helper API

Shorthand for common vault field patterns with sensible defaults (exclude: true, l10n_mode: exclude).

use Netresearch\NrVault\TCA\VaultFieldHelper;

'api_key' => VaultFieldHelper::getSecureFieldConfig(
    'API Key',
    ['required' => true]
),

Security Features

Envelope Encryption

Each secret is encrypted with its own unique Data Encryption Key (DEK), which is then encrypted with the master key.

Access Control

Secrets can be restricted to specific backend users or groups. Every access is logged in the audit trail.

Audit Logging

Tamper-evident audit chain records all secret operations with cryptographic integrity verification.
